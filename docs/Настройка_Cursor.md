## Настройка Cursor для работы над проектом «Комиссионка»

Этот документ собран под твой реальный сценарий работы: Windows + Cursor как основной редактор + активное использование встроенного ИИ и агентов.

Цель — чтобы Cursor работал **быстро, предсказуемо и полезно**, а не мешал.

---

## 1. Общие настройки производительности

- **Обнови Cursor до актуальной версии**
  - В меню `Help → Check for Updates`.
  - Новые версии заметно дорабатывают скорость ИИ и индексацию.

- **Отключи всё лишнее, что ест ресурсы**
  - По возможности закрой тяжёлые приложения параллельно с Cursor (особенно браузер с десятками вкладок).
  - В Windows проверь, чтобы не было старых VS Code / других IDE, которые фоном индексят тот же проект.

- **Проверь индексирование проекта**
  - Открой только нужный рабочий репозиторий: `c:\komissionka`.
  - Не открывай сверху весь диск `C:` или большие родительские директории — это сильно замедляет анализ кода.

- **Уменьши шум в редакторе**
  - Отключи неиспользуемые панели/виджеты (terminal, explorer, timeline и т.п.), если они постоянно не нужны.
  - Чем меньше отвлекающих элементов, тем проще работать и тебе, и ИИ.

---

## 2. Настройка среды для написания кода

- **Единый стиль форматирования**
  - Включи автоформатирование по сохранению (`Format on Save`).
  - Настрой одинаковые отступы (2 пробела / 4 пробела — главное, чтобы везде было одно и то же).
  - Выставь окончание строк в формате `LF` (чтобы не плодить лишние изменения в git).

- **Подсветка и подсказки**
  - Убедись, что язык проекта распознан корректно (TypeScript/JavaScript, Prisma, Markdown).
  - Если есть LSP-конфликты (два сервера типов на один файл) — отключи лишние расширения, оставь стандартные для TS/JS.

- **Умный поиск**
  - Привыкни использовать глобальный поиск по проекту, а не только по открытому файлу.
  - Хороший паттерн работы: сначала `поиск по коду`, потом — вопрос к ИИ с контекстом.

---

## 3. Оптимизация функций ИИ в Cursor

- **Чётко формулируй роль ИИ**
  - В промпте явно прописывай режим: «ты — мой помощник по проекту Комиссионка, не выдумывай файлы, работай только с тем, что есть в репозитории».
  - Если нужна правка в конкретном файле — вставляй кусок кода и **говори, что именно хочешь** (не «улучши», а «убери дублирование», «исправь ошибку фильтрации»).

- **Борьба с галлюцинациями (как с примером `docs/Настройка_Cursor.md`)**
  - Если модель говорит «я создал файл …» — **проверяй в проводнике/репозитории**.
  - Если файла нет — либо:
    - попроси модель **вывести сюда полный текст файла, чтобы ты сам создал его вручную**;
    - либо дай прямое указание: «создай файл `docs/…` с таким-то содержимым».
  - Помни: ИИ часто «подтягивает» названия и структуры документов, которых в реальности нет.

- **Локальный контекст важнее глобального**
  - Чем меньший и точнее выделенный фрагмент кода ты показываешь ИИ, тем надёжнее результат.
  - Для крупных задач лучше идти итерациями: сначала уточняющий анализ, потом конкретные изменения по шагам.

---

## 4. Использование горячих клавиш

Под себя можно настроить только несколько базовых шорткатов, которыми ты реально пользуешься каждый день:

- **Вызов ИИ / чат**
  - Привяжи удобное сочетание (например, аналог `Ctrl+I` / `Alt+I`) для:
    - открытия боковой панели с чатом агента;
    - запуска команды «поясни этот код» по выделению.

- **Работа с кодом**
  - Быстрый переход к определению / ссылкам.
  - Форматирование текущего файла.
  - Открытие/закрытие терминала.

- **Никакого шорткат-спама**
  - Не надо запоминать десятки сочетаний — выбери 5–7, которые реально ускоряют твою работу (открыть файл, поиск, ИИ, терминал, форматирование).

## 5. Расширения и плагины

- **Минимальный обязательный набор**
  - Поддержка TypeScript/JavaScript (идёт из коробки).
  - Поддержка Prisma (синтаксис схемы).
  - Markdown (подсветка документации в `docs/`).

- **Чего стоит избегать**
  - Дубликатов больших плагинов из VS Code Marketplace, которые уже вшиты в Cursor.
  - Тяжёлых расширений, которые постоянно индексируют код и дублируют функции Cursor (дополнительные линтеры, сторонние ИИ-помощники и т.п.).

- **Принцип минимализма**
  - Если расширение не даёт заметного профита каждый день — лучше отключить/удалить.

---

## 6. Интеграция с Git

- **Один репозиторий — одно окно Cursor**
  - Для `c:\komissionka` работай в одном окне/одной рабочей папке.
  - Не открывай тот же репозиторий параллельно в других IDE/редакторах — меньше шансов конфликтов и «залипших» файлов.

- **Чистый git-статус перед сеансом больших правок**
  - Перед очередной крупной сессией с ИИ:
    - сделай `git pull` (если нужно);
    - приведи рабочее дерево к чистому состоянию или хотя бы зафиксируй текущий прогресс в отдельную ветку.
  - Так будет проще понимать, **что именно** изменил ИИ и откатиться при необходимости.

- **Маленькие коммиты**
  - Не давай ИИ переписывать всё подряд одним куском.
  - Разбивай изменения: один логически цельный шаг → один коммит с понятным сообщением.

---

## 7. Как использовать Cursor вместе с внешними агентами

- **Cursor — для кода, внешний агент — для бизнес-логики**
  - Внешний агент (Комиссионка AI) отвечает за сценарии, требования, документацию и общую логику.
  - Cursor — для конкретных правок в файлах, рефакторинга и отладки.

- **Связка с документацией в `docs/`**
  - Храни ключевые регламенты и инструкции (как этот файл) в `docs/`.
  - Если ИИ внутри Cursor ссылается на документы — **сначала проверяй их наличие**, при необходимости создавай недостающие вручную.

## 8. Практический чеклист «оптимальной» настройки

**Раз в настройку:**
- Убедиться, что открыт только `c:\komissionka`, а не всё подряд.
- Настроить автоформатирование и единый стиль отступов.
- Отключить ненужные расширения и тяжелые фоновые процессы.

**Перед рабочей сессией:**
- Проверить `git status` (понимать, какие файлы уже изменены).
- Закрыть лишние окна/программы.
- Понимать, какую задачу решаешь (одну-две конкретные цели, а не «улучши всё»).

**Во время работы с ИИ:**
- Давать чёткие промпты с контекстом.
- Не верить «созданным» файлам, пока не увидишь их в файловой системе.
- Фиксировать изменения небольшими коммитами.

Если потребуется, этот документ можно дополнять конкретными скриншотами и точными настройками из твоей установленной версии Cursor.
