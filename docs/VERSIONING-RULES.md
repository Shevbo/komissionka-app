# Правила версионности компонентов

Формат версии: **X.Y.Z** (строгая глобальная логика)

## Логика инкремента

- **X+1 (мажор)** — фича/изменение, затронувшее **>30% строк** в core-модулях компонента
- **Y+1 (минор)** — фича/задача, затронувшая **<30% строк** в core
- **Z+1 (патч)** — исправление бага, мелкое улучшение (для текущей X.Y)

**Важно:** при инкременте X или Y — **Z сбрасывается в 0**. Например: 1.2.3 → 2.0.0 (мажор) или 1.2.3 → 1.3.0 (минор).

## Запрет

**Модель не может менять версии по запросу пользователя** («обнови версию», «сделай версию»). Версии меняются **только** когда внесённые изменения соответствуют логике выше. Если пользователь просит «обнови версии» без фактических изменений — отказ.

## Core-модули (для расчёта %)

| Компонент | Core-модули |
|-----------|-------------|
| **App** | `src/app/`, `src/lib/`, `prisma/schema.prisma` |
| **Agent** | `agent/core.ts`, `agent/contract.ts`, `agent/serve.ts`, `agent/config.ts`, `agent/llm/`, `agent/tools/`, `agent/cache/` |
| **Tgbot** | `telegram-bot/bot.ts` |

## Распознавание задания

Перед версионированием явно определи: **фича** (новая функциональность → Y+1) или **баг** (ошибка, сбой → Z+1). В подвале отчёта указывай: «Задание распознано как: фича(и)» и/или «баг(и)».

## Деплой — обязательно самостоятельно

После изменений кода модель выполняет деплой сама: `.\scripts\deploy-hoster.ps1 -Upload -Restart`. Не писать «убедитесь, что задеплоено».

## Обязательные действия при изменении кода

1. **Обновить** `version.json` (и `package.json` при наличии) для затронутых компонентов по логике X.Y.Z
2. **Добавить запись** в [what's new.md](what's new.md) (корень) по структуре:
   - 1) Дата время > запрос пользователя
   - 2) Дата время > по каждому: 2.1 Комиссионка (web), 2.2 Агент, 2.3 Телеграм-Бот, 2.4 PWA — описание, новые объекты, заменено N из M строк core, документы [ссылки], версия была → стала
3. **Деплой** — выполнить самому `.\scripts\deploy-hoster.ps1 -Upload -Restart`
4. **В конце отчёта** — подвал: «Задание распознано как: …», версии до/после, таблица дат ключевых файлов (из `.\scripts\server-key-files-dates.ps1`)
5. **Подсчёт строк** — `npx tsx scripts/count-core-lines.ts`

## Откат

При откате — восстановить `version.json` и `what's new.md`.
