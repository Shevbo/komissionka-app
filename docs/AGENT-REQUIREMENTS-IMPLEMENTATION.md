# Реализация требований к агенту (agent_req.txt)

## 1. Универсальный агент для множества проектов

- **Параметр `project`** добавлен в вызов агента (по умолчанию «Комиссионка»).
- Настройка через `AGENT_PROJECT` в `.env` или в `agent/config.json`.
- Админка и Telegram-бот передают `project: "Комиссионка"` в запросе.
- В body POST `/run` можно указать `project`, `userAccount`, `chatName`, `environment`.

## 2. Локальное кэширование промптов и ответов

- **Таблица `agent_prompt_cache`** в PostgreSQL хранит:
  - дату/время, пользователя, модель, проект, среду, режим, чат, тему, промпт, ответ;
  - ссылки на файлы, кол-во слов (вход/выход), хэш промпта.

- **Проверка сходства** перед вызовом LLM: функция `textSimilarity()` (0–100%).
- **Порог** `AGENT_CACHE_SIMILARITY_THRESHOLD=70` — при совпадении ≥70% возвращается кэш вместо вызова модели.
- **Обход кэша**: префикс `! ` или слово `force` в промпте — принудительный вызов LLM.
- **Ограничение размера** `AGENT_CACHE_MAX_BYTES` (по умолчанию 4 ГБ). При превышении удаляются старые записи.

## 3. Команды @agent:

В чате с агентом:

- `@agent: все темы` — список тем порциями по 20.
- `@agent: тема «*API*»` — промпты и ответы по теме, содержащей «API».
- `@agent: промпт «*сбой*»` — записи, где в промпте есть «сбой».
- `@agent: экспорт (проект=«Комиссионка» & тема=«*api*» & (промпт=«*сбой*» | ответ=«*сбой*»))` — экспорт в CSV.

## 4. Браузер кэша в админке

- Новая вкладка **«Кэш промптов»** в админке.
- Фильтры: проект, тема, промпт.
- Копирование промпта/ответа в буфер.
- Экспорт выбранного фильтра в CSV.

## 5. Режим «курилка»: write_file только для docs

- Добавлен инструмент **`write_docs_file(path, content)`**.
- Разрешена запись только в папку `docs/` (например `docs/инструкция.md`).
- Модель информирована в системном промпте.
