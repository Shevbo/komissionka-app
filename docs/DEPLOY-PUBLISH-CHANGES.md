# Публикация изменений на внешний хостинг

Порядок действий для новичка: как отправить изменения из рабочего репозитория на сервер. Команды можно вводить в терминале и на сервере по SSH.

---

## Что нужно заранее

- Проект хранится в **Git** (есть рабочий репозиторий).
- Код выкладывается на **GitHub / GitLab / другой Git-хостинг** (у вас есть репозиторий и доступ).
- На **внешнем сервере** проект уже один раз развёрнут: папка с кодом (клонирована из Git), установлены Node.js, настроены `.env`, PM2 и Nginx (см. `HOSTING-RU-MIGRATION.md`).

Дальше каждый раз при публикации изменений делаете два шага: **1) в рабочем репозитории** и **2) на сервере**.

---

## Часть 1. Команды в рабочем репозитории

Откройте **терминал в Cursor**: меню **Terminal → New Terminal** или `` Ctrl+` ``.

### Шаг 1. Проверить, что вы в папке проекта

```bash
cd ~/komissionka
```

### Шаг 2. Посмотреть, что изменилось

```bash
git status
```

Будут видны изменённые или новые файлы.

### Шаг 3. Добавить все изменения в индекс

```bash
git add .
```

(Точка = все изменения в текущей папке.)

### Шаг 4. Создать коммит с коротким описанием

```bash
git commit -m "Описание изменений: например, правка баннера и корзины"
```

Вместо текста в кавычках пишите своё описание на русском или английском.

### Шаг 5. Отправить изменения на Git-хостинг

Если ветка называется `main`:

```bash
git push origin main
```

Если ветка называется `master`:

```bash
git push origin master
```

Имя ветки видно в строке после `git status` (например, `On branch main`).

Если Git спросит логин и пароль — введите данные от вашего аккаунта на GitHub/GitLab. После успешного `git push` код уже на удалённом репозитории.

---

## Часть 2. Команды на сервере (после подключения по SSH)

Подключитесь к серверу так, как настроили при первом развёртывании, например:

```bash
ssh root@IP_ВАШЕГО_СЕРВЕРА
```

(Подставьте свой IP или имя пользователя вместо `root`, если используете другого пользователя.)

Дальше выполняйте команды **уже на сервере**.

### Шаг 1. Перейти в папку проекта

```bash
cd /home/nextjs-app
```

(Замените путь на тот, где у вас лежит проект, например `/var/www/komissionka`.)

### Шаг 2. Забрать новые изменения из Git

```bash
git pull origin main
```

Если ветка у вас `master`, то:

```bash
git pull origin master
```

Если Git запросит логин/пароль — введите их или настройте на сервере SSH-ключ для доступа к репозиторию.

### Шаг 3. Установить зависимости (если менялся package.json)

```bash
npm install
```

### Шаг 4. Prisma: сгенерировать клиент и применить миграции

Если менялась схема БД (`prisma/schema.prisma` или папка `prisma/migrations`):

```bash
npx prisma generate
npx prisma migrate deploy
```

Если схема БД не менялась — эти команды можно пропустить.

### Шаг 5. Собрать приложение

```bash
npm run build
```

### Шаг 6. Перезапустить приложение (PM2)

```bash
pm2 restart komissionka
```

Если при первом запуске вы дали другое имя процессу — подставьте его вместо `komissionka`. Список процессов: `pm2 list`.

### Шаг 7. Проверить, что приложение запустилось

```bash
pm2 status
```

Должно быть `online`. Логи при необходимости:

```bash
pm2 logs komissionka
```

Выход из логов: `Ctrl+C`.

## Краткая шпаргалка (копировать по шагам)

**В рабочем репозитории:**

```bash
cd ~/komissionka
git status
git add .
git commit -m "Ваше описание изменений"
git push origin main
```

**На сервере (после `ssh ...`):**

```bash
cd /home/nextjs-app
git pull origin main
npm install
npx prisma generate
npx prisma migrate deploy
npm run build
pm2 restart komissionka
pm2 status
```

---

## Если репозиторий ещё не привязан или нет ветки main

Один раз нужно настроить удалённый репозиторий (подставьте свой URL):

```bash
git remote add origin https://github.com/ВАШ_ЛОГИН/komissionka.git
```

Проверить имя ветки:

```bash
git branch
```

Если текущая ветка не `main`, в командах выше везде используйте её имя (например, `master`).

---

## Если на сервере проект пока не клонирован из Git

Один раз на сервере:

```bash
cd /home
git clone https://github.com/ВАШ_ЛОГИН/komissionka.git nextjs-app
cd nextjs-app
```

Дальше создайте `.env`, выполните `npm install`, `npx prisma generate`, `npx prisma migrate deploy`, `npm run build`, настройте PM2 и Nginx по инструкции в `HOSTING-RU-MIGRATION.md`. После этого для публикации изменений используйте уже порядок из этого файла (pull → install → prisma → build → pm2 restart).
