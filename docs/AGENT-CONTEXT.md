# Контекст проекта для агента

Этот файл подставляется в системный промпт агента (если задан `AGENT_CONTEXT_FILE=docs/AGENT-CONTEXT.md`). Здесь — краткий «опыт» по проекту: что уже есть, как устроено, куда смотреть.

---

## Проект «Комиссионка»

- Веб-приложение: маркетплейс комиссионных товаров (Next.js, Prisma, PostgreSQL).
- Стек: Next.js 16, React 19, Prisma 7, NextAuth, Tailwind/shadcn, Zustand.
- Репозиторий: корень = рабочая директория агента; пути к файлам — относительно корня.

## Структура

- `src/` — код приложения (app router, компоненты, lib, API).
- `prisma/` — схема БД и сиды.
- `docs/` — документация (миграции, бот, Prisma, этапы агента и т.д.).
- `agent/` — локальный AI-агент (Вариант C): контракт, инструменты (read_file, list_dir, find_files, grep, run_command), ядро с LLM и циклом вызовов инструментов.

## Что уже сделано по агентскому пайплайну

- Этап 1: контракт (stdin/stdout), конфиг (AGENT_*), окружение.
- Этап 2: инструменты чтения и поиска, grep, run_command с белым списком; описания для LLM.
- Этап 3: клиент LLM (OpenAI-совместимый), системный промпт, цикл рассуждений с лимитом шагов, таймаут и обработка ошибок.

## Версионность

- `version.json` — версии app, agent, tgbot (X.Y.Z).
- `docs/VERSIONING-RULES.md` — правила: при любом изменении обновлять версии и `what's new.md` затронутого компонента; в конце отчёта — блок версий до/после.

## Полезные документы

- `docs/VERSIONING-RULES.md` — правила версионности (обязательно при изменениях).
- `docs/TELEGRAM-BOT-VARIANT-C-STAGES.md` — этапы Варианта C (агент в репозитории, затем API для бота).
- `docs/PRISMA-CLI-WORKFLOW.md` — работа с Prisma.
- `docs/HOSTING-RU-MIGRATION.md` — миграция/хостинг.

## Запуск агента

Из корня: `echo "промпт" | npm run agent:start`. Конфиг: `.env` с переменными `AGENT_*` или `agent/config.json`.
