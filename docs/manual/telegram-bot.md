## 2. Архитектура

Бот — отдельный процесс, запускаемый через PM2. Работает в цикле long polling, принимает сообщения от пользователей Telegram.

Потоки данных:

1. **Привязка профиля** — пользователь вводит код из ЛК → бот обновляет `profiles.telegram_id`, `profiles.telegram_username`.
2. **Запрос к агенту** — текст пользователя → POST к агент API → ответ (текст + картинки) → отправка в Telegram.
3. **Управление** — кнопки «Модель ИИ», «Режим», «Проект», «Последний ход» → вызов API приложения.

```
telegram-bot/
└── bot.ts    # Единственный файл — логика бота
```

## 3. Функциональные блоки

### 3.1 Точка входа

| Файл | Описание |
|------|----------|
| [telegram-bot/bot.ts](../../telegram-bot/bot.ts) | Создание экземпляра `TelegramBot`, обработчики `on("message")`, `on("callback_query")` |

### 3.2 Инициализация

- `TELEGRAM_BOT_TOKEN` — обязателен.
- `APP_BASE_URL` — URL приложения (https://komissionka92.ru или http://127.0.0.1:3000).
- `AGENT_PORT`, `AGENT_API_KEY` — для вызова агента.
- Обработка `polling_error` (409 Conflict — токен занят).

### 3.3 Привязка профиля

| Логика | Описание |
|--------|----------|
| Код привязки | Пользователь вводит код из ЛК (страница с /api/admin/telegram-bind-code). |
| Проверка | Запрос к `GET /api/telegram/bind?code=...&telegram_id=...` (или POST). |
| Обновление | Приложение обновляет `profiles.telegram_id`, `profiles.telegram_username` по коду. |
| Только привязанные | Запросы к агенту принимаются только от привязанных профилей. |

### 3.4 Вызов агента

| Этап | Описание |
|------|----------|
| Конфиг | `fetchAgentConfig()` — GET /api/agent/selected-model (модель, режим). |
| Запрос | POST к `http://127.0.0.1:AGENT_PORT/run` с `{ prompt, history?, mode, modelOverride, inputImages? }`. Для сообщений с фото бот скачивает изображение и передаёт его как data URL в `inputImages`. |
| Заголовок | `Authorization: Bearer AGENT_API_KEY`. |
| Парсинг ответа | `parseResultSegments()` — разбивка на текст и изображения `![alt](url)`. |
| Отправка | `sendResultSegments()` — текст по 4096 символов, картинки — sendPhoto (локальный файл, URL или base64). |

### 3.5 Управление (кнопки)

| Кнопка | API | Действие |
|--------|-----|----------|
| 🤖 Модель ИИ | /api/telegram/agent-models, /api/telegram/set-model | Список моделей, выбор модели (callback). |
| ⚙️ Режим | /api/telegram/agent-mode, /api/telegram/set-mode | Текущий режим, смена (chat/consult/dev). |
| 📁 Проект | /api/telegram/bot-context | Список проектов (если несколько). |
| 📋 Последний ход | /api/telegram/agent-log | Последний лог рассуждений агента. |

### 3.6 Картинки в ответах

- `/api/uploads/agent/<filename>` — доступ к картинкам, сгенерированным агентом.
- Локальный путь: `public/uploads/agent/<filename>`.
- Поддержка `data:image/...;base64,...` и `http(s)://` URL.
- Входные картинки: если админ отправляет боту сообщение с фото (с подписью или без), бот скачивает изображение через Telegram API, кодирует в `data:image/jpeg;base64,...` и передаёт в агент как `inputImages`. Модели Gemini получают картинку как inlineData и могут анализировать её содержимое.

### 3.7 Ход выполнения

`formatSteps()` — форматирование массива `steps` от агента в читаемый текст для отправки в Telegram (по запросу «Последний ход» или при длительном ответе).

## 4. Модель данных

Бот использует модели приложения:

| Модель | Использование |
|--------|---------------|
| `profiles` | telegram_id, telegram_username — привязка по коду |
| `telegram_bind_code` | code, profile_id, expires_at — коды привязки |

Данные о запросах к агенту не сохраняются в БД ботом — агент сам пишет в `agent_prompt_cache`.

## 5. Доступ к Prisma и БД

Бот импортирует `prisma` из `src/lib/prisma` и использует его при необходимости (например, проверка привязки). Основная логика привязки — в API приложения.

Подключение к БД: см. [docs/DB-MANUAL-INSPECT.md](../DB-MANUAL-INSPECT.md) и [app-komissionka.md](app-komissionka.md#5-доступ-к-prisma-и-бд).

**Запуск бота на сервере:**

```bash
npm run bot   # или через PM2: pm2 start bot
```

Переменные: `TELEGRAM_BOT_TOKEN`, `APP_BASE_URL`, `AGENT_PORT`, `AGENT_API_KEY`.

## 6. Deploy: от разработки до продакшена

### 6.1 Условная схема: от разработки до внедрения в прод

```
[Разработка] → [Коммит] → [Загрузка на сервер] → [Применение на сервере]
     │              │              │                        │
     │              │              │                        └─ pm2 restart bot
     │              │              └─ rsync / incremental scp / full scp
     │              └─ version.json, what's new.md (tgbot)
     └─ telegram-bot/ — изменение кода (bot.ts)
```

**Этапы:**

1. **Разработка** — правки в `telegram-bot/bot.ts` (и при необходимости в API приложения).
2. **Версионирование** — обновление `version.json` (tgbot), блок UPDATE в `what's new.md`.
3. **Загрузка** — синхронизация всего репозитория. Инкрементальная синхронизация передаёт только изменённые файлы.
4. **Применение** — на сервере: `pm2 restart bot` (в составе общего деплоя — `pm2 restart komissionka agent bot`).

### 6.2 Месторасположение изменённого кода (Dev)

| Компонент | Каталог / файлы |
|-----------|-----------------|
| Бот | `telegram-bot/bot.ts` |
| API приложения (привязка, режимы) | `src/app/api/telegram/` |

Корень репозитория — каталог с `package.json`. Бот — подкаталог `telegram-bot/`.

### 6.3 Целевое расположение на сервере (Prod)

| Элемент | Путь на сервере |
|---------|-----------------|
| Репозиторий (включая telegram-bot/) | `~/komissionka` |
| Код бота | `~/komissionka/telegram-bot/` |
| Запуск | PM2: процесс `bot` (`npm run bot:start`) |

Сервер: VPS 83.69.248.175.

### 6.4 Инструкция по инкрементальному и полному развёртыванию

**Скрипт деплоя:** [scripts/deploy-hoster.ps1](../../scripts/deploy-hoster.ps1)

Бот разворачивается вместе с приложением и агентом — общий скрипт синхронизирует весь репозиторий.

**Полный деплой:**

```powershell
.\scripts\deploy-hoster.ps1 -All
```

**Инкрементальная синхронизация** — как у приложения: rsync или PowerShell по размеру файлов; при сбое — полный scp.

**Только перезапуск бота** (код уже на сервере):

```bash
ssh hoster "cd ~/komissionka && pm2 restart bot"
```

**Полный цикл вручную:**

```bash
ssh hoster "cd ~/komissionka && pm2 restart komissionka agent bot"
```
