# Как деплоить (инструкция для Cursor / агента в другом чате)

Версия: 1.0 (сервис деплоя v1.13.0)

## Главное

- **Код и всё окружение — на сервере.** VPS 83.69.248.175, каталог `~/komissionka`. Локальной разработки нет.
- **Деплой выполняешь сам** после любых изменений кода. Не проси пользователя «задеплоить».

## Как задеплоить (один способ)

1. **Закоммить и запушить** изменения в `main` (если ещё не сделано).
2. **Запустить деплой** из корня проекта (Windows PowerShell):
   ```powershell
   .\scripts\deploy-hoster-git.ps1
   ```
   По умолчанию деплоится среда **prod** (основное приложение на порту 3000).
3. Скрипт сам: проверит чистое дерево git, сделает `git push origin main`, добавит задачу в очередь деплоя. Обработчик на сервере (deploy-worker) выполнит `env-deploy.sh` для prod в течение 1–5 минут.
4. **Резервный вариант** (если API очереди недоступен): прямой SSH-деплой:
   ```powershell
   .\scripts\deploy-hoster-git.ps1 -NoQueue
   ```

## Деплой в тестовую среду (test1, test2, …)

- Та же команда, с указанием среды:
  ```powershell
  .\scripts\deploy-hoster-git.ps1 -Env test1
  ```
- Тестовые среды живут в `~/komissionka-test1`, `~/komissionka-test2` и т.д., с отдельными портами и БД.

## Вкладка «Деплой» в админке

- Админка: **https://komissionka92.ru/admin** (или http://83.69.248.175:3000/admin). Нужна роль **admin**.
- Вкладка **«Деплой»** показывает:
  - **Среды** — prod, test1, … (статус, порт, кнопки «Деплой», «Открыть», «Удалить» для не-prod).
  - **Очередь операций** — текущие и недавние задачи (pending/running/completed). Если очередь пуста — это нормально после завершения всех операций.
  - **Журнал операций** — история create/deploy/copy/delete с логами и длительностью.
- Если видишь «Нет доступа к API деплоя» — войди в админку под учёткой с ролью admin и обнови страницу (Ctrl+F5).
- Данные обновляются автоматически раз в 10 секунд; кнопка «Обновить» под заголовком «Среды» запрашивает данные сразу.

## Где что лежит

| Что | Где |
|-----|-----|
| Скрипт деплоя (основной) | `scripts/deploy-hoster-git.ps1` |
| Резервный деплой (scp) | `scripts/deploy-hoster.ps1` |
| Серверный деплой prod | На сервере: `~/komissionka/scripts/deploy-from-git.sh` |
| Серверный деплой/создание сред | `~/komissionka/scripts/env-deploy.sh`, `env-create.sh`, `env-copy.sh`, `env-delete.sh`, `env-status.sh` |
| Обработчик очереди | `scripts/deploy-worker.ts` (PM2: deploy-worker) |
| API деплоя | `src/app/api/deploy/*` |
| UI вкладки Деплой | `src/components/DeployTab.tsx` |
