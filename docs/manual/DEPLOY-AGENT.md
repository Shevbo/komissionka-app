# Как деплоить (инструкция для Cursor / агента в другом чате)

Версия: 1.1 (пошаговая инструкция, вкладка Деплой, резерв и test1)

## Главное

- **Код и всё окружение — на сервере.** VPS 83.69.248.175, каталог `~/komissionka`. Локальной разработки нет.
- **Деплой выполняешь сам.** От пользователя ничего не просят: ни коммит, ни «что включить в коммит», ни деплой. Свои изменения коммитишь и пушишь сам, затем запускаешь деплой. Пользователь только смотрит статусы в админке.

---

## Пошагово: как задеплоить прод

1. **Закоммить и запушить** изменения в `main` (если ещё не сделано).
2. Из **корня проекта** (каталог с `package.json`, например `c:\komissionka-v2`) в PowerShell выполнить:
   ```powershell
   .\scripts\deploy-hoster-git.ps1
   ```
   Или явно прод и ветка: `.\scripts\deploy-hoster-git.ps1 -Branch main` (среда prod по умолчанию).
3. Скрипт сам: проверит чистое дерево git → `git push origin main` → отправит задачу в очередь (POST `/api/deploy/queue`). Обработчик на сервере (**deploy-worker**) выполнит `env-deploy.sh prod main` в `~/komissionka` в течение 1–5 минут (fetch, npm ci, prisma, build, pm2 restart).
4. Если в консоли появилось **«Failed to queue via API, falling back to direct SSH»** — скрипт сам переключился на прямой SSH и запустил деплой на сервере; дождись окончания (обновление кода, npm install, build, pm2 restart).

## Резервный вариант (прямой SSH без очереди)

Если очередь часто недоступна или нужно гарантированно выполнить деплой без API:

```powershell
.\scripts\deploy-hoster-git.ps1 -NoQueue
```

Скрипт сразу выполнит на сервере `deploy-from-git.sh` (prod) или `env-deploy.sh` (test1, …) по SSH, без постановки в очередь.

## Деплой в тестовую среду (test1, test2, …)

- Та же команда с параметром среды:
  ```powershell
  .\scripts\deploy-hoster-git.ps1 -Env test1
  ```
- Тестовые среды: каталоги `~/komissionka-test1`, `~/komissionka-test2` и т.д., отдельные порты и БД. Среда должна быть создана заранее (вкладка «Деплой» → «Создать среду» или скрипты `env-create.sh` на сервере).

## Куда смотреть в админке (вкладка «Деплой»)

- Админка: **https://komissionka92.ru/admin** (или http://83.69.248.175:3000/admin). Нужна роль **admin**.
- Вкладка **«Деплой»** показывает:
  - **Среды** — prod, test1, … (статус, порт, кнопки «Деплой», «Открыть», «Удалить» для не-prod).
  - **Очередь операций** — текущие и недавние задачи (pending/running/completed).
  - **Журнал операций** — история create/deploy/copy/delete с логами и длительностью.
- **Журнал всех операций** — все операции без пропусков: по очереди (админка, Cursor, агент ИИ) и прямые запуски скриптов (deploy-from-git.sh, env-deploy.sh). Колонки: дата, среда, операция, статус, источник (очередь/скрипт), кто запросил, длительность. Чтобы прямые запуски попадали в журнал, на сервере в `~/komissionka/.env` должна быть переменная **DEPLOY_LOG_SECRET** (любая строка-секрет); скрипты при завершении отправляют запись в API с этим секретом.
- **Пустая очередь после выполнения операций — норма.** Активность и результат смотри в **«Журнал операций»** (последние строки — последний деплой).
- Если видишь **«Нет доступа к API деплоя»** — войди в админку под учёткой с ролью admin и обнови страницу (Ctrl+F5).
- Данные обновляются автоматически раз в 10 секунд; кнопка «Обновить» под заголовком «Среды» запрашивает данные сразу.

## Где что лежит

| Что | Где |
|-----|-----|
| Скрипт деплоя (основной) | `scripts/deploy-hoster-git.ps1` |
| Резервный деплой (scp) | `scripts/deploy-hoster.ps1` |
| Серверный деплой prod | На сервере: `~/komissionka/scripts/deploy-from-git.sh` |
| Серверный деплой/создание сред | `~/komissionka/scripts/env-deploy.sh`, `env-create.sh`, `env-copy.sh`, `env-delete.sh`, `env-status.sh` |
| Обработчик очереди | `scripts/deploy-worker.ts` (PM2: deploy-worker) |
| API деплоя | `src/app/api/deploy/*` |
| UI вкладки Деплой | `src/components/DeployTab.tsx` |
